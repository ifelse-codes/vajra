# Session 18 — Product Review + Vision Building

## Goal
Walk through everything Vajra has built, test each command manually, collect feedback, brainstorm what's next, and shape the product vision + roadmap.

## Type
INTERACTIVE — not a formal ground truth audit, not a code session. A product review with the founder.

## What to cover

### 1. Hands-on walkthrough (test each command in a real or scratch repo)
- `vajra init` — scaffold from scratch, inspect output
- `vajra check` — run drift detection, inspect results
- `vajra next` — print packet, try `--advance`
- `vajra estimate` — see the cost prediction
- `vajra claude` — launch a real session (short), see compression + receipt
- `vajra meter` — inspect a past session's cost

### 2. Collect feedback along the way
- What feels right?
- What's confusing or ugly?
- What's missing?
- What would you show someone in a demo?

### 3. Brainstorm + vision building
- What should Vajra be in 6 months?
- What's the first thing a new user should experience?
- Where does the product go after Claude is solid?
- Update VISION.md, ROADMAP.md, and backlog based on findings

## Constraints
- No code changes unless the user explicitly asks for a quick fix during the review
- Branch: `session-18-product-review-and-vision`
- Outputs: updated ROADMAP.md, updated VISION.md (if needed), brainstorm notes in session summary
