The Chaos of AI Coding

Cross-Session Amnesia

Agents forget context between sessions, repeating work, breaking state, and losing the thread of complex tasks.

Silent Drift

Subtle deviations from the architecture accumulate unnoticed until the codebase is unrecognizable and brittle.

Autonomous Bad Commits

Without mechanical barriers, agents commit broken code, bypass verification, and pollute the git history.

Token Bloat

Re-reading entire codebases and verbose states wastes context windows, slows down the agent, and burns money.

Defense in Depth

Vajra enforces discipline through 5 mechanical layers. It doesn't ask the AI to be good; it makes bad behavior mechanically impossible.

L1

L4
Integer State (Single Source of Truth in .ai/)

Core Mechanics

Branch-per-Session

Every session operates on an isolated session-NN-<slug> branch. No more stepping on your own toes.

Fail-Closed Verification

verify-closeout.sh must exit 0. If it fails, the session cannot close. No silent passes.

Ground-Truth Audits

Every N sessions, a hook-enforced, no-code audit forces a full drift and staleness check.

Approval-Gated Commits

Mechanical barriers require explicit approval tokens before autonomous commits are allowed.

Delta-Only Boot

Agents don't re-read the world. They read .ai/DELTA—only what changed since the last session.

Cross-Agent Compatibility

Root pointers (AGENTS.md, CLAUDE.md, .cursorrules) all route to a single source of truth.

Forged for Speed

~90%

Fewer tokens per session.

Token efficiency isn't a feature; it's a consequence of rigorous design. By using hash-cached constitutions, silent hooks, and delta-only boots, Vajra eliminates redundant context loading. Zero quality compromised.
